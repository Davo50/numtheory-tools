Лекция 1. Числовые методы криптографии — краткий конспект

Представление чисел и длина записи, сложность базовых операций, \(\mathbb{Z}_N\), алгоритм Евклида и Безу, цепные дроби, быстрое возведение в степень, Карацуба, алгоритм Монтгомери (по слайдам).

Обозначения: \(\mathbb{Z}_N\), \(\gcd\), \(L_p(\cdot)\) Модель: асимптотики \(O(\cdot)\) по длине входа Ключевые подзадачи: mod-умножение, \(x^e\bmod N\), обратный по модулю

1) Системы счисления и длина записи числа

Определение Позиционная \(p\)-ичная запись. Любое целое \(x \ge 0\) представляется как \[ x=\sum_{i=0}^{n-1} x_i p^i,\quad 0\le x_i < p,\quad x_{n-1}\ne 0 \ (x>0). \]
Определение Длина записи \(L_p(x)\). Число цифр в \(p\)-ичной записи \(x>0\). Эквивалентно: \[ p^{L_p(x)-1}\le x < p^{L_p(x)}. \]
Оценка Длина произведения. Для \(x,y>0\): \[ L_p(xy) \le L_p(x)+L_p(y), \] и снизу верно (с точностью до константы): \[ L_p(x)+L_p(y)-O(1)\le L_p(xy). \]
В тетради сдвиг может выглядеть как «\(-2\)» из-за выбора границ вида \(p^{n-1}\le x\le p^n\).

2) Стоимость базовых операций над целыми

Идея Сложность выражается через длину входа (количество цифр/бит).
Утверждение Сложение/вычитание. \[ T_{+/-}=O(\max(L_p(x),L_p(y))). \]
Утверждение Умножение (школьное). \[ T_{\times}=O(L_p(x)\,L_p(y)). \]
Идея из конспекта Деление «в столбик». Для \(n\)-разрядного на \(m\)-разрядное: порядка \(n-m+1\) шагов, в каждом — действия на \(m\) разрядах, т.е. оценка вида \(O(m\,(n-m+1))\) на уровне «разрядных операций».

3) Арифметика по модулю: \(\mathbb{Z}_N\)

Определение \(\mathbb{Z}_N\) — кольцо вычетов по модулю \(N\). Операции: \(x\pm y\bmod N\), \(xy\bmod N\).
Приведение Для произведения обычно пишут: \[ r \equiv xy \pmod N,\qquad r = xy - qN \] для некоторого частного \(q\) (на практике — деление/редукция).
Оценки (битовые) Пусть \(N\) — \(n\)-битное (то есть \(L_2(N)\approx n\)). Тогда:
  • \(x\pm y \bmod N\): \(O(n)\).
  • \(xy \bmod N\) при школьном умножении: \(O(n^2)\).
Ключевая подзадача Нахождение обратного \(x^{-1}\pmod N\) (решается расширенным Евклидом при \(\gcd(x,N)=1\)).

4) Алгоритм Евклида, НОД, Безу. Теорема Ламе

Алгоритм Евклид. Для \(a\ge b>0\): \[ a=bq_1+r_1,\quad b=r_1q_2+r_2,\ \dots,\ r_{k-2}=r_{k-1}q_k+r_k,\quad r_k=0. \] Тогда \(\gcd(a,b)=r_{k-1}\).
Теорема (Безу) Существуют целые \(x,y\), что \[ ax+by=\gcd(a,b). \] Следствие: если \(\gcd(a,N)=1\), то коэффициент \(x\) даёт \[ a^{-1}\equiv x\pmod N. \]
Алгоритм Расширенный Евклид. Вместе с остатками поддерживаются коэффициенты так, чтобы на каждом шаге выполнялось \(r_i = ax_i + by_i\). В конце получаем \(\gcd(a,b)=ax+by\).
Теорема Ламе (о числе шагов Евклида) Если алгоритм Евклида для \((a,b)\), \(a\ge b\), делает \(k\) делений (шагов), то \(b \ge F_{k+1}\), где \(F_i\) — числа Фибоначчи. Поэтому число шагов ограничено логарифмом: \[ k \le \log_{\varphi}(b) + O(1) = O(\log b), \] где \(\varphi=\frac{1+\sqrt5}{2}\).
Худший случай достигается на соседних числах Фибоначчи.

5) Цепные дроби (непрерывные дроби)

Определение \[ \alpha = q_0 + \cfrac{1}{q_1 + \cfrac{1}{q_2 + \cdots}} \] (конечная цепная дробь для рациональных, бесконечная — для иррациональных).
Связь с Евклидом Частные \(q_i\) — это частные в делениях алгоритма Евклида.
Сходящиеся Сходящиеся \(\dfrac{P_k}{Q_k}\) удовлетворяют рекурсиям: \[ P_k = q_k P_{k-1} + P_{k-2},\qquad Q_k = q_k Q_{k-1} + Q_{k-2}. \]
Начальные условия можно задавать эквивалентными способами (важна сама рекурсия).

6) Быстрое возведение в степень по модулю

Алгоритм Square-and-multiply. Для вычисления \(x^e \bmod N\) используем двоичную запись \(e=\sum e_i2^i\), затем последовательно выполняем возведения в квадрат и умножения по модулю.
Оценка Число mod-умножений порядка \(O(L_2(e))\), т.е. \[ T \approx O(L_2(e)\cdot M(n)), \] где \(M(n)\) — стоимость умножения \(n\)-битных чисел (школьно \(M(n)=O(n^2)\)).

7) Умножение Карацубы. Мастер-теорема

Идея Разбить \(x,y\) на половины (для двоичной базы): \[ x = x_1 2^k + x_0,\qquad y = y_1 2^k + y_0, \] и вычислить произведение через 3 рекурсивных умножения: \[ x_1y_1,\quad x_0y_0,\quad (x_1+x_0)(y_1+y_0), \] вместо 4.
Рекурсия Для \(n\) (размер/длина): \[ T(2n)\le 3T(n)+cn. \] Следовательно, \[ T(n)=O\!\left(n^{\log_2 3}\right)\approx O(n^{1.585}). \]
Мастер-теорема (шпаргалка) Для рекурсии \[ T(n)=a\,T\!\left(\frac{n}{b}\right)+f(n),\quad a\ge 1,\ b>1, \] положим \(n^{\log_b a}\) как «базовый» рост. Типовые случаи (достаточно для Карацубы и похожих оценок):
  • Если \(f(n)=O\!\left(n^{\log_b a-\varepsilon}\right)\), то \(T(n)=\Theta\!\left(n^{\log_b a}\right)\).
  • Если \(f(n)=\Theta\!\left(n^{\log_b a}\log^k n\right)\), то \(T(n)=\Theta\!\left(n^{\log_b a}\log^{k+1} n\right)\).
  • Если \(f(n)=\Omega\!\left(n^{\log_b a+\varepsilon}\right)\) и выполнено условие регулярности \(a f(n/b)\le c f(n)\) для некоторого \(c<1\), то \(T(n)=\Theta(f(n))\).

8) Алгоритм Монтгомери (по слайдам): определение, теорема, алгоритмы

Условия (со слайда) Выбираем \(R\) так, что
  • \(R>N\) и \(\gcd(R,N)=1\);
  • известно представление (со слайда): \[ RR' - NN' = 1,\qquad 0
На практике часто берут \(R=2^k\), чтобы деление на \(R\) было сдвигом.
Монтгомери-умножение (со слайда) Вводится новая операция \(\otimes\) на \(\mathbb{Z}_N\): \[ x\otimes y \equiv xyR \pmod N. \] И отображение \[ \varphi_R:\mathbb{Z}_N\to\mathbb{Z}_N,\qquad \varphi_R(x)=xR^{-1}\pmod N. \]
Со слайда: задача — доказать, что \(\varphi_R\) является изоморфизмом колец \((\mathbb{Z}_N,+,\cdot)\) и \((\mathbb{Z}_N,+,\otimes)\). Проверка умножения: \(\varphi_R(xy)=\varphi_R(x)\otimes \varphi_R(y)\).
Теорема (со слайда) Для любого целого \(0\le x < RN\) и числа \[ m \equiv xN' \pmod R \] выполняются свойства для \[ t=\frac{x+mN}{R}: \]
  1. \(t\) — целое число;
  2. \(0\le t < 2N\);
  3. \(t \equiv xR^{-1}\pmod N\).
Это и есть «Montgomery reduction»: вычисление \(xR^{-1}\bmod N\) без деления на \(N\).
Algorithm (со слайда) Вычисление \(\varphi_R(x)\) (Montgomery reduction). 
Вход:  x ∈ Z_N   (обычно x трактуют как целое 0 ≤ x < RN)
Выход: φ_R(x)

1) m = x·N' (mod R)
2) t = (x + m·N) / R
3) if 0 ≤ t < N:  return t
4) else:          return t - N
Algorithm (со слайда) Умножение по модулю через Монтгомери.
Идея: сначала посчитать «монтгомери-произведение» \(z=\varphi_R(xy)=xyR^{-1}\bmod N\), затем вернуться в обычное представление применением \(\varphi_R^{-1}\). 
Вход:  x, y ∈ Z_N
Выход: x·y (mod N)

1) z  = φ_R(x·y)       = x·y·R^{-1} (mod N)
2) xy = φ_R^{-1}(z)
3) return xy
Примечание: на практике обычно работают в «монтгомери-форме» (числа хранятся как xR mod N), чтобы на каждом шаге не делать явного φ_R^{-1}.
Algorithm (со слайда) Возведение в степень (через монтгомери-умножение \(\varphi\)). 
Вход:  x ∈ Z_N, k ∈ N
Выход: x^k (mod N)

1) y1 = φ_R^{-1}(x) = x·R (mod N)
2) Для i = 2..k:
       yi = φ(yi-1, y1) = x^i·R (mod N)    // умножение в монтгомери-форме
3) return φ_R(yk) = x^k·R·R^{-1} (mod N) = x^k (mod N)
Здесь \(\varphi(\cdot,\cdot)\) — «Montgomery multiply», возвращающее результат в монтгомери-форме.
Algorithm (со слайда) \(p\)-ичный вариант (сканирование по цифрам), где \(R=p^n\).
На слайде: пусть \(L_p(N)=n\), \((N,p)=1\), \(R=p^n\). Тогда можно вычислять \(\varphi_R(xy)=xyR^{-1}\bmod N\) итеративно по цифрам \(x\) в базе \(p\). 
Вход:  x, y ∈ Z_N,  x = Σ_{i=0}^{n-1} x_i p^i
Выход: z_n ≈ φ_R(xy)

1) z0 = 0
2) Для i = 1..n:
       u = (z_{i-1} + x_{i-1}·y) (mod p)
       v = u·N' (mod p)
       z_i = (z_{i-1} + x_{i-1}·y + v·N) / p
3) if 0 ≤ z_n < N:  return z_n
4) else:            return z_n - N

Итог лекции (коротко)

  • Длина \(L_p(x)\) связывает величину числа и стоимость базовых алгоритмов.
  • Евклид и расширенный Евклид дают НОД, коэффициенты Безу и обратный по модулю; по Ламе число шагов \(O(\log b)\).
  • Быстрое \(x^e\bmod N\) сводится к последовательности mod-умножений; ускорение умножения: Карацуба (\(O(n^{\log_2 3})\)).
  • Монтгомери позволяет делать редукцию/умножение по модулю эффективно (особенно при \(R=2^k\)), что ускоряет modular exponentiation.
Если надо — могу (а) добавить мини-доказательство изоморфизма \(\varphi_R\), (б) сделать один полностью разобранный числовой пример Montgomery reduction на маленьких \(N,R\), (в) привести единый блок «сложности» в терминах \(M(n)\) (школьное/Карацуба).
