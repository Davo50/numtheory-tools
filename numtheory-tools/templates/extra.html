🧮 Многочлены F₂[X] и Монтгомери

Бинарный алгоритм Евклида для многочленов в F₂[X] (адаптация псевдокода Л1 стр. 18 к многочленам). Вход — два многочлена в битовой записи (старший коэффициент слева): например 10011101 = x⁷+x⁴+x³+x²+1. В F₂[X]: вычитание = XOR, «делится на 2» ↔ свободный коэф равен 0 (⇔ x | f). Для корректной работы на многочленах сравнение «a ≥ b» заменено на сравнение по степени.
Алгоритм Карацубы для F₂[X] (Л1 стр. 33–34). Формула препода: xy = (X²ᵏ+Xᵏ)·x₁y₁ ⊕ (x₁⊕x₀)(y₁⊕y₀)·Xᵏ ⊕ (Xᵏ+1)·x₀y₀. Сложность T(2n) ≤ 3T(n) + cn ⇒ T(n) = O(n^log₂3) ≈ O(n^1.585).
Умножение по Монтгомери (Л1 стр. 36–40). По определению: x ⊗ y = x·y·R⁻¹ (mod N), где R > N, НОД(R, N) = 1. Алгоритм вычисляет φ_R(xy): m = xy·N' mod R; t = (xy + m·N)/R; если t ≥ N, возвращаем t − N, иначе t. R выбирается как минимальная степень 2, большая N (если не задано иное).
Пример: 10011101 = x⁷+x⁴+x³+x²+1
Пример: 111110 = x⁵+x⁴+x³+x²+x

Нужно R > N и НОД(R, N) = 1.
