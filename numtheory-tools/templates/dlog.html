🔐 Дискретный логарифм

Алгоритм Гельфонда-Шэнкса (Baby-step Giant-step), Л5 стр. 4. Считаем s = ⌈√n⌉, строим малый шаг S = {(y·g^r, r) : r=0…s−1} и большой шаг T = {(g^ts, ts) : t=0…s}, ищем совпадение. Тогда x = ts − r. Сложность O(√n · log n).
ρ-алгоритм Полларда для ДЛ, Л5 стр. 18–20. Разбиение G = A ∪ B ∪ C по остатку z mod 3. Параллельно считаем z_i и z_{2i}, ждём коллизию. Тогда x = (α_{2i} − α_i) · (β_i − β_{2i})⁻¹ mod n. Преимущество — память O(1).
Алгоритм Полига-Хеллмана, Л5 стр. 9–16. Работает, когда p−1 — B-гладкое число. Раскладываем p−1 = ∏ pᵢ^αᵢ, для каждой силы pᵢ^αᵢ находим x mod pᵢ^αᵢ по таблицам r_{pᵢ,j} = g^(j·(p−1)/pᵢ), потом склеиваем ответ по Китайской теореме об остатках.
Это число из дано после знака «=». Например, в задаче «Дано: g = 5, g^x = 12, p = 23, найти x» — сюда вводишь 12.
Программа сама посчитает g^x mod p и потом найдёт x обратно — для проверки работы алгоритма.
Примеры: g=5, g^x=12, p=23 (BSGS) g=3, g^x=24, p=31 (BSGS, ответ 13) g=2, g^x=62, p=181 (Полиг, ответ 100) g=6, g^x=13, p=229 (ответ 117) — для Index calc (на /advanced)
