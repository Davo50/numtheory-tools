# numtheory-tools

Учебные инструменты по теоретико-числовым методам криптографии. Веб-приложение на Flask с интерактивными, пошаговыми реализациями классических алгоритмов — факторизация, НОД, дискретный логарифм, арифметика в F₂[X], умножение по Монтгомери, эллиптические кривые.

Каждый алгоритм выводит все промежуточные шаги — удобно для учёбы, подготовки к контрольным и просто чтобы «увидеть, как это работает».

---

## Что внутри

**Факторизация:** Ферма, p−1 Полларда, ρ Полларда, Ленстра (ECM), квадратичное решето
**НОД:** классический Евклид, бинарный Евклид, НОД нескольких чисел, расширенный Евклид (Безу)
**Цепные дроби:** разложение рационального числа, подходящие дроби P_s/Q_s
**Быстрое возведение в степень:** бинарный метод
**Дискретный логарифм:** BSGS (Гельфонда-Шэнкса), ρ Полларда, Полиг-Хеллман, Index Calculus (Адлеман)
**F₂[X]:** бинарный Евклид для многочленов, алгоритм Карацубы
**Модульная арифметика:** умножение по Монтгомери
**Эллиптические кривые:** сложение точек, удвоение, скалярное умножение, порядок точки, ECDLP через BSGS

В каждой странице есть примеры одним кликом — значения подставятся, и можно сразу запускать.

---

## Как запустить

Есть два способа — выбирай по вкусу.

### Способ 1 — без Docker (обычный Python)

Требуется **Python 3.9+**. Проверить версию:

```bash
python3 --version
```

Дальше:

```bash
# 1. Перейти в папку проекта
cd numtheory-tools

# 2. (опционально, но желательно) создать виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows

# 3. Поставить зависимости
pip install -r requirements.txt

# 4. Запустить
python app.py
```

В терминале увидишь:

```
  ┌─ numtheory-tools ─────────────────────────────────────────────
  │  Локально:  http://127.0.0.1:8080   http://localhost:8080
  │  По сети:   http://192.168.X.Y:8080
  │             (с телефона/другого ПК в той же Wi-Fi)
  └──────────────────────────────────────────────────────────
```

Открой **первую ссылку** в браузере. Вторая — чтобы зайти с телефона или другого устройства в той же Wi-Fi (нужно быть в одной сети + на самом компьютере может понадобиться разрешить входящие подключения в настройках брандмауэра).

Остановить: `Ctrl+C` в терминале.

### Способ 2 — через Docker

Требуется **Docker Desktop** (для Mac/Windows) или **docker engine + docker compose** (Linux).

```bash
cd numtheory-tools

# Собрать образ и запустить в фоне
docker compose up --build -d

# Посмотреть логи (увидишь те же два адреса)
docker compose logs -f

# Остановить
docker compose down
```

После `docker compose up` сервер доступен по `http://localhost:8080`. Адрес по локальной сети — как у хост-машины (узнать: `ipconfig getifaddr en0` на Mac, `ipconfig` на Windows, `hostname -I` на Linux), порт тот же 5000.

Docker автоматически перезапустит контейнер после ребута (`restart: unless-stopped`), так что один раз поставил — и пользуешься.

## Если что-то не работает

- **Port 8080 is in use** — у тебя уже что-то крутится на этом порту (на Mac часто занят AirPlay Receiver: System Settings → General → AirDrop & Handoff → выключить). Или поменяй порт в `app.py` (строка `port = 8080`) и `docker-compose.yml`.
- **При запуске показывается странный IP вида `198.18.0.1`, `100.x.x.x` или `10.x.x.x`** и с других устройств не работает — это **не твой реальный LAN-адрес**, его раздаёт VPN-клиент (Cloudflare Warp, AnyConnect, Proton VPN, ClearVPN, Surfshark и т.п.) или Tailscale. Что делать:
  1. Отключи VPN (закрой приложение или поставь на паузу).
  2. Узнай реальный Wi-Fi IP: `ipconfig getifaddr en0` (если пусто — попробуй `en1`). Настоящий домашний адрес — `192.168.x.x` или реже `10.x.x.x` (но не `10.64.*`, не `100.64.*`, не `198.18.*`).
  3. Если отключить VPN нельзя, смотри строку «Другие интерфейсы» в выводе `python app.py` — там будет и правильный адрес.
- **Не могу зайти с телефона по правильному LAN-адресу** — проверь, что оба устройства в одной Wi-Fi (не «Гостевой»), и что на компьютере разрешены входящие подключения. macOS: System Settings → Network → Firewall → Options → разрешить python. Windows: Windows Defender Firewall → новое правило на порт 8080.
- **LAN IP не найден** — значит сеть недоступна в момент старта. Локально (`http://127.0.0.1:8080`) работать всё равно будет.
- **Ошибки импорта sympy/flask** — забыл `pip install -r requirements.txt` или не активировал venv.

## Структура проекта

```
1/
├── app.py                       # Flask-приложение, роуты и API
├── requirements.txt             # зависимости Python
├── Dockerfile                   # сборка Docker-образа
├── docker-compose.yml           # конфиг для docker compose up
├── .dockerignore
├── quadratic_sieve_verbose.py   # отдельный модуль QS
├── algos/                       # все алгоритмы
│   ├── euclid.py                # 4 варианта Евклида + цепные дроби
│   ├── fastpow.py               # быстрое возведение в степень
│   ├── dlog.py                  # BSGS, ρ Полларда, Полиг-Хеллман
│   ├── index_calculus.py        # Index Calculus (Адлеман)
│   ├── elliptic.py              # эллиптические кривые + ECDLP
│   ├── poly_f2.py               # многочлены в F₂[X]
│   ├── montgomery.py            # умножение по Монтгомери
│   ├── default_factor.py        # общие функции факторизации
│   └── quadratic_sieve.py       # старая SageMath-версия QS (не используется)
├── templates/                   # HTML-страницы
│   ├── index.html               # факторизация
│   ├── gcd.html                 # НОД
│   ├── tools.html               # Безу / CF / степень
│   ├── dlog.html                # дискретный log
│   ├── advanced.html            # index calc + EC
│   ├── extra.html               # F₂[X] / Монтгомери
│   └── notes.html                 # конспект Л1
└── static/                      # MathJax (рендер формул в /notes)
```

Каждый модуль в `algos/` содержит `if __name__ == '__main__'` с self-тестами на эталонах из лекций. Чтобы прогнать:

```bash
python -m algos.euclid
python -m algos.dlog
python -m algos.index_calculus
python -m algos.elliptic
# и т.д.
```
